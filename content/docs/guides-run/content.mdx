**Run → Content** is the full content lifecycle on one surface: from idea to a
published WordPress post. It moves through stages you can see as tabs.

## The lifecycle

<Steps>
<Step>
### Plan
Work from your content strategy and the **Opportunities** the app surfaces
(content gaps and refresh candidates).
</Step>
<Step>
### Brief
Create a **brief** for a target keyword. Pagerise can generate an editorial brief
— search intent, a suggested outline, secondary keywords, audience questions, and
a target word count — so the draft has a real plan behind it.
</Step>
<Step>
### Draft
Generate the article. Drafts are written for search intent **and** AI answer
engines, grounded in the client's [Brain](/docs/guides-run/brain) for true
E-E-A-T, with internal links pulled from the site's crawl index. You can pick the
per-site writing model in Settings → Branding.
</Step>
<Step>
### Schedule
Set a publish date to queue the post for the future, or publish now.
</Step>
<Step>
### Publish
Push to WordPress in one click. Pagerise handles the post, slug, meta title and
description, featured image, and (where applicable) schema.
</Step>
</Steps>

## Drafts and review

Generated drafts land in the **Drafts** stage for review. You can edit, approve,
reject, regenerate, or publish. Published posts move to the **Published** stage
and link out to the live URL.

<Callout type="info">
  **Refresh in place:** for a page that already ranks, Pagerise can update the
  live post (title/meta or a full body rewrite) instead of creating a duplicate —
  preserving the URL.
</Callout>

## Tips

- Add real client facts to the [Brain](/docs/guides-run/brain) before generating —
  it's the difference between generic copy and copy that reflects the client.
- Use **Opportunities** to prioritize refreshes of pages stuck on page two.
