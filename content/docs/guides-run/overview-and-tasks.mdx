**Run → Overview** is the home screen for a site; **Run → Tasks** is the
prioritized list of everything worth fixing.

## Overview

The Overview gives you the state of the site at a glance:

- A KPI strip — traffic, AI visibility, and Site Health.
- The **Site Health** gauge with category breakdowns.
- The **AI Visibility** hero (latest score and engine coverage).
- GSC / GA4 trends, WordPress status, recent jobs, and crawl history.
- A content-calendar snapshot and links to the latest reports.

## Tasks

**Tasks** is the fix-it console: every issue and opportunity the audit found,
across all categories, ranked by severity. It carries a live open-count chip in
the nav so you always know how much is outstanding.

Each task explains the problem and why it matters. Many are **one-click
fixes** — when a site is connected to WordPress, Pagerise can apply the change
(for example, writing a missing meta description or injecting schema) directly to
the live page.

<Steps>
<Step>
### Triage by severity
Start at the top — critical and high-severity issues move the needle most.
</Step>
<Step>
### Apply or assign
Apply one-click fixes where available, or work the rest manually.
</Step>
<Step>
### Re-crawl to confirm
Run a fresh crawl so resolved issues drop off and the health score updates.
</Step>
</Steps>

<Callout type="info">
  Tasks is generated from the latest crawl snapshot. If a fix isn't reflected,
  re-crawl the site.
</Callout>
