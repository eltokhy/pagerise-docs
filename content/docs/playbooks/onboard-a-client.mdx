---
title: Onboard a client
description: A complete client setup in Pagerise — from adding the domain to delivering the first report.
---

This playbook takes a brand-new client from zero to a first deliverable.

<Steps>
<Step>
### Create the client & site
Add the client, then add their site (domain). See
[Connect your first site](/docs/getting-started/connect-first-site).
</Step>
<Step>
### Connect data sources
Connect **GSC** and **GA4** for performance data, **WordPress** for publishing,
and **GBP** if it's a local business.
[Integrations →](/docs/settings/integrations)
</Step>
<Step>
### Crawl & audit
Run the first [crawl and audit](/docs/getting-started/first-crawl-and-audit).
Work the critical/high [Tasks](/docs/guides-run/overview-and-tasks).
</Step>
<Step>
### Fill the Brain
Add the client's real expertise, results, and brand facts to the
[Brain](/docs/guides-run/brain) so content reflects them.
</Step>
<Step>
### Baseline AI visibility
Run an [AI Visibility](/docs/guides-track/ai-visibility) audit to set a baseline
and get the recommended playbook.
</Step>
<Step>
### Plan content
Turn keyword gaps and opportunities into [Content](/docs/guides-run/content)
briefs; for local businesses, generate [Local Pages](/docs/guides-run/local-pages).
</Step>
<Step>
### Deliver the first report
Generate and [share a branded report](/docs/guides-report/sharing-and-branding)
so the client sees the starting point and the plan.
</Step>
</Steps>

<Callout type="info">
  Do steps 4–5 (Brain + AI baseline) before generating any content — both make the
  output meaningfully better.
</Callout>
